    mood (1-5), exercised (bool), sleep_hours, notes

    user_id, domain (grades/health/life), action, xp,
    difficulty (easy/medium/hard), reason, completed (bool)

--------------------------------------------------------------------------------
ARCHITECTURE — HOW THE APP WORKS
--------------------------------------------------------------------------------

  1. User signs up / logs in via Supabase Auth
  2. User completes onboarding form → saved to user_profiles table
  3. Canvas integration pulls course data → saved to canvas_courses table
  4. User submits daily checkin → saved to checkin_logs table
  5. User requests suggestions → Python backend fetches profile + canvas +
     checkins → passes to Claude API → saves and returns structured JSON
  6. Frontend displays suggestions as game cards with XP and difficulty

  Frontend reads from Supabase directly for simple data (profile, history).
  Frontend always goes through the Python backend for anything involving Claude.

--------------------------------------------------------------------------------
API ENDPOINT REFERENCE
--------------------------------------------------------------------------------

  POST /onboarding        Save user profile after onboarding form
  POST /checkin           Submit a daily checkin entry
  POST /suggestions       Generate AI suggestions for the user
  POST /canvas/sync       Sync latest Canvas data for the user

  All endpoints require a Bearer token in the Authorization header.
  The backend validates the token and extracts user_id automatically.
